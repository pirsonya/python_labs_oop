## Описание интерфейсов

В рамках работы создано два абстрактных базовых класса (интерфейса):

### `Printable` (интерфейс вывода)
Требует реализации метода `to_string() -> str`, который возвращает строковое представление объекта

### `Comparable` (интерфейс сравнения)
Требует реализации метода `compare_to(other) -> int`, который сравнивает текущий объект с другим:
- возвращает **-1**, если текущий объект "меньше"
- возвращает **0**, если объекты равны
- возвращает **1**, если текущий объект "больше"

## Реализация в классах

На основе классов из предыдущих лабораторных работ (`Author`, `Translator`, `Biographer`) созданы классы-обёртки, реализующие интерфейсы

### Какие классы реализуют интерфейсы

- `ComparableAuthor` — реализует оба интерфейса (Printable и Comparable)
- `ComparableTranslator` — реализует оба интерфейса
- `ComparableBiographer` — реализует оба интерфейса

### Как отличается поведение

**Метод to_string() из интерфейса Printable:**

- `ComparableAuthor` — выводит: "Author: Имя Фамилия, books: число"
- `ComparableTranslator` — выводит: "Translator: Имя Фамилия, translations: число, lang pair: язык->язык"
- `ComparableBiographer` — выводит: "Biographer: Имя Фамилия, subjects: темы, interviews: True/False"

Метод compare_to() из интерфейса Comparable:

- `ComparableAuthor` — сравнивает только по году рождения
- `ComparableTranslator` — сначала сравнивает по количеству переводов, если одинаково — по году рождения
- `ComparableBiographer` — сначала сравнивает по количеству биографий, если одинаково — по году рождения
