# ЛР-1 — Класс и инкапсуляция 
## Предметная область - Библиотека/Книги (Вариант 2); реализованный класс - Author
#### Цель работы: Освоить объявление пользовательских классов, разобраться с инкапсуляцией (атрибуты экземпляра, закрытые поля), реализовать свойства (@property), переопределить магические методы (__str__, __repr__, __eq__), осознать разницу между атрибутами класса и экземпляра.
### Краткое описание процесса
Реализованный класс - Author (автор). Характеристики объекта: имя, фамилия, год рождения, год смерти, страна рождения, количество написанных книг и жанр, в котором автор пишет произведения. Добавлена валидация (валидация на тип данных, логику). Логика проверок вынесена в отдельный модуль validate.py для удобства тестирования и возможного переиспользования в других классах.

Также добавлены бизнес-методы и магические методы. Бизнес-методы способствуют быстрому нахождению информации об авторе (получить полную информацию, раскрыть список написанных книг, получить кол-во написанных книг, обновить/добавить n-ое кол-во книг). Добавлены магические методы для удобной работы с объектом. Создан демонстрационный файл, показывающий все возможности класса.

### Краткое описание класса
class Author содержит поля: 

- _name (имя)
- _surname (фамилия)
- _birth_year (год рождения)
- _death_year (год смерти)
- _country (страна рождения)
- _genre (жанр, в котором пишет автор)
- _count_books (количество написанных книг)
- _is_active (активен/неактивен)

### Атрибут класса
_genre_categories = 
["роман", "поэзия", "ужасы", "комедия", "детектив", "драма"]

### Методы класса
- 'def get_full_name' - получить полное ФИО
- 'def get_initials' -получить инициалы
- 'def is_alive' - узнать жив/мертв
- 'def get_age_in_year' - узнать возраст 
- 'def get_current_age' - узнать текущий возраст (даже если автор мертв)
- 'def get_life_period' - получить годы жизни
- 'def add_book' - добавить книгу
- 'def update_count_books' - обновить кол-во книг
- 'def get_info' - получить полную информацию об авторе

### Магические методы
- __init__ - инициализатор
- __repr__ - техническое представление объекта
- __str__ - строковое представлние 
- __eq__ - сравнение объектов

### Методы для изменения логического состояния
- def activate - активировать автора 
- def deactivate - деактивировать автора

(активация/деактивация автора - им.в.в. изменение состояния биографических карточек авторов в электронной системе библиотеке)

### Свойства @property (сеттеры/геттеры)
Геттер (Getter) — метод, который получает значение атрибута.

Сеттер (Setter) — метод, который устанавливает значение атрибута.

Для каждого закрытого атрибута реализованы свойства с геттерами и сеттерами, обеспечивающие валидацию при изменении значений. В случае данной лабораторной работы был использован режим доступа protected (взаимодействие с данными из класса и/или из дочерних классов).

### модуль validate.py
- def validate_name - проверка имени (тип данных)
- def validate_year - проверка года рождения/смерти (логическое состояние, должно быть целым числом, не меньше min и не больше max-current year)
- def validate_country - проверка страны рождения (тип данных)
- def validate_genre - проверка жанра (тип данных)
- def validate_count_books - проверка кол-ва книг (тип данных, целое число, положительное число)

все функции были вынесены в отдельный модуль (файл) validate.py
